单列集合set，双列集合map

set（接口）存储的无序且不可重复的数据

Set中方法都是从Collection继承过来的，没有特有方法

 无序性：  数据在存储时并非按照数组索引顺序进行存储的，是由存储数据的哈希值决定的
不可重复性： 在添加数据时调用hashCode、equals方法，返回值不能为true

HashSet添加元素底层实现原理

在HashSet中添加元素a，首先调用元素a对象的hashCode方法获取其哈希值
根据哈希值通过某种算法，找到需要在数组中存储的位置（引索），如果该位置还没有存储元素，那么元素a添加成功（情况1）
如果该位置存储了元素b，那么就比较两者的哈希值是否相等，如果两种哈希值一致，那么元素a添加成功（情况2）
如果哈希值一致，那么就调用equals方法比较两个对象：
(1).如果equals()方法返回值是true，则认为两个对象是同一个对象，元素a添加失败（情况3）
(2).如果equals()返回false，则元素a添加成功（情况4）

对于情况1、4，元素a，元素b在同一个位置是使用链表的结构存储的

  • JDK1.7 新增元素a保存在数组中位置，a元素指向原有元素b
  • JDK1.8 原有元素b依旧还在数组位置上，该元素指向元素a

总结：HashSet底层结构：数组+链表
如果需要向HashSet中存储数据，如果需要实现（相同属性数据无法存在多个），需要重写hashCode、equals方法

每个元素维护了两个引用，一个指向上一个元素，一个指向下一个元素。
linkHashSet更适合做set集合的遍历。

Java中比较器

Java中存在比较运算符：> 、< 、>= 、<= 、!= 、== 、instanceof
只能比较基本数据类型，对于对象无法比较，如果在对象中需要实现对象间的比较，只能通过Comparable、Comparator
1.Comparable接口
String、包装类都已经默认实现了Comparable接口，并重写了int CompareTo(T o)方法，所以String、包装类都
默认是按照升序进行排序的
int compareTo(T o)如果返回值是正数，代表调用者(this)的值比o要大。负数比o小，0则相等。
如果一个自定义类需要实现排序功能，需要让当前类实现Comparable接口，重写compareTo（T o）方法，在该方法中重写排序的规则
如果一个类一旦实现了Comparable接口，那么这个类的对象在任意地方都可以进行排序或比较

2.Comparator接口（定制排序）
如果一个类没有实现Comparable接口，但是这个类又不方便实现Comparable接口，或者一个类已经实现了Comparable接口，但是其中的compareTo不满足我们的需求，那么
可以使用Comparator接口方式实现定制排序
默认升序
int compare(T o1, T o2);
o1>o2 返回正数
o1<o2 返回负数
o1==o2 返回0

package com.greedring;

import java.util.Arrays;
import java.util.Comparator;

public class Test {
    public static void main(String[] args) {
        String s1 = "A";
        String s2 = "B";
        //System.out.println(s1>s2);

        String[] arr = new String[]{"AA", "BB", "DD", "FF", "EE", "KK"};
        int[] arr2 = new int[]{1, 6, 10, 6, 8, 11, 100, 98};
        Arrays.sort(arr);
        Arrays.sort(arr2);
        System.out.println(Arrays.toString(arr));
        System.out.println(Arrays.toString(arr2));

        Student[] users = new Student[5];
        users[0] = new Student(2, "Jack");
        users[1] = new Student(1, "Rose");
        users[2] = new Student(1, "Jerry");
        users[3] = new Student(1, "Tom");
        users[4] = new Student(3, "Marry");
        //Arrays.sort(users);//com.igeekhome.pm.User cannot be cast to java.lang.Comparable
        //System.out.println(Arrays.toString(users));
        Integer i1 = 12;
        int compareTo = i1.compareTo(20);
        System.out.println(compareTo);

        //按照id进行升序，如果id重复按照姓名降序(定制排序)
        Arrays.sort(users, new Comparator() {

            @Override
            public int compare(Object o1, Object o2) {
                if (o1 instanceof Student && o2 instanceof Student) {
                    Student s1 = (Student) o1;
                    Student s2 = (Student) o2;
                    if (s1.getId() > s2.getId()) {
                        return 1;
                    } else if (s2.getId() < s2.getId()) {
                        return -1;
                    } else {
                        //id如果相同按照name倒序
                        return -s1.getName().compareTo(s2.getName());
                    }
                }
                return 0;
            }
        });

        System.out.println(Arrays.toString(users));
    }

    public static class Student implements Comparable {

        private Integer id;
        private String name;

        Integer getId() {
            return id;
        }

        public void setId(Integer id) {
            this.id = id;
        }

        String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }

        Student(Integer id, String name) {
            super();
            this.id = id;
            this.name = name;
        }

        public Student() {
            super();
        }

        @Override
        public int hashCode() {
            final int prime = 31;
            int result = 1;
            result = prime * result + ((id == null) ? 0 : id.hashCode());
            result = prime * result + ((name == null) ? 0 : name.hashCode());
            return result;
        }

        @Override
        public boolean equals(Object obj) {
            if (this == obj)
                return true;
            if (obj == null)
                return false;
            if (getClass() != obj.getClass())
                return false;
            Student other = (Student) obj;
            if (id == null) {
                if (other.id != null)
                    return false;
            } else if (!id.equals(other.id))
                return false;
            if (name == null) {
                if (other.name != null)
                    return false;
            } else if (!name.equals(other.name))
                return false;
            return true;
        }


        @Override
        public String toString() {
            return "Student [id=" + id + ", name=" + name + "]";
        }

        @Override
        public int compareTo(Object o) {
            if (o instanceof Student) {
                Student s = (Student) o;
                //按照id升序
                /*
                 * if(this.getId()>s.getId()) { return 1; }else if(this.getId()<s.getId()) {
                 * return -1; }else { return 0; }
                 */
                //借助Integer内部的compareTo方法
                return this.getId().compareTo(s.getId());

            }
            return 0;
        }
    }
}

class MyUserComparator implements Comparator<User> {

    @Override
    public int compare(User o1, User o2) {
        if (o1.getId().compareTo(o2.getId()) != 0) {
            return o1.getId().compareTo(o2.getId());
        } else {
            return -o1.getName().compareTo(o2.getName());
        }
    }
}

Map

接口，双列集合（存储一对一对的数据）类似函数y=f(x)
|—- HashMap:Map的主要实现类，线程不安全，效率高，可以存储为null的key和value
|—- LinkedHashMap：HashMap的子类，能够按照元素的添加顺序进行遍历（因为每个元素包含了两个引用Entry<K,V> before, after;分别指向上一个元素以及下一个元素）
如果涉及到频繁遍历操作时，LinkedHashMap性能高于HashMap
|—- TreeMap:可以对添加到集合中的元素进行排序（按照key的自然排序或者如果key是对象的话，按照对象的指定属性进行排序【自然排序、定制排序】）
|—- Hashtable:古老的双列集合，线程安全的（同步方法），效率低 不可以存储为null的key和value
|—- Properties:操作属性配置文件的，key、value都必须是String类型（IO）

Map、key、value的特点
以HashSet为例
Key是无序且不可重复，如果需要获取一个Map的所有的Key，可以使用的是Set结构
Value是无序可重复的，如果需要获取一个Map的所有Value，可以使用Collection结构存储
一个Map中的Entry对象无序且不可重复的，如果需要获取一个Map的所有键值对(Entry)，可以使用Set结构

Hash底层实现原理
HashMap map = new HashMap();
当创建HashMap时，底层默认创建一个Entry数组，并进行默认初始化，初始化长度为16

//之前已经存储了部分key…value
map.put(k1,v1)
先调用k1对象的hashCode方式，，根据k1的哈希值通过某种算法，得到元素需要再底层数组存储的位置（索引0-）
判断当前索引位置上是否有元素，如果当前索引位置上没有元素，k1，v1直接添加成功[情况1]
判断当前索引位置上存在一个或多个元素，将k1的哈希值依次与这一个或多个已经存在的key的哈希值进行比较，如果k1的哈希值与已存在元素的哈希值都不同，k1、v1添加成功[情况2]
如果k1的哈希值与已经存在的元素中的任意一个元素的key的哈希值相同(k2,v2)
调用equals方法比较k1、k2，如果返回false，k1、v1添加成功[情况3]
如果equals方法返回true，那么v1值将覆盖v2[情况4]

1.对于情况2、3多个元素再同一个数组中的位置使用链表进行存储
2.对于put过程（添加元素），可能会涉及到扩容操作，当数组使用长度超过临界值（而并非是数组长度），并且当前数组位置不为空的时候，需要进行扩容，扩容到原来的2倍，将将原有集合中数据拷贝到新的集合中

术语（常量）
DEFAULT_INITIAL_CAPACITY，默认初始化长度 16
DEFAULT_LOAD_FACTOR 默认加载因子 0.75（计算临界值（threshold）=16*0.75=12）
TREEIFY_THRESHOLD JDK1.8 桶中节点数量树化最大值（链表->红黑树）
MIN_TREEIFY_CAPACITY：64 树化数组的最小长度

JDK1.8 与 JDK 1.7区别：
（1）JDK1.7中底层是Entry[]数组 JDK1.8中底层是Node<K,V>[] table
static class Node<K,V> implements Map.Entry<K,V> {
final int hash;
final K key;
V value;
Node<K,V> next;
(2)JDK1.7中当调用HashMap()构造函数时，底层会创建一个长度为16的Entry数组，而JDK1.8中当调用构造函数时，并不会初始化底层数组
当第一次调用put操作时，初始化长度为16的Node[] 数组
（3）JDK1.7中桶中的数据是新增数据指向原有数据，JDK1.8是原有数据指向新增数据
（4）JDK1.8在某种情况（当前数组索引下节点的长度超过了8并且数组长度超过了64）下，会将桶中（数组中某个索引位置）的数据由链表结构转换为树形结构（树化）

从这个映射中移除所有的映射（可选操作）。

如果涉及到频繁遍历操作时，LinkedHashMap性能高于HashMap

LinkedHashMap 在 HashMap基础之上，每个节点对象
static class Entry<K,V> extends HashMap.Node<K,V> {

Entry(int hash, K key, V value, Node<K,V> next) {
super(hash, key, value, next);
}
}

4.static <T extends Object & Comparable> T max(Collection coll) 返回最大元素的集合，根据其元素的自然排序。 5.static T max(Collection coll, Comparator comp) 返回给定集合的最大元素，根据指定的比较器诱导的顺序。 6.static > T min(Collection coll) 返回最小的元素的集合，根据其元素的自然排序。 7.static T min(Collection coll, Comparator comp) 返回给定集合的最小元素，根据指定的比较器诱导的顺序。 8.static void reverse(List list)
反转指定列表中元素的顺序。
9.static void shuffle(List list) 随机置换指定列表使用随机默认源。 10.static > void sort(List list) 指定列表为升序排序，根据其元素的 natural ordering。 11.static void sort(List list, Comparator c) 根据指定的比较器指定的顺序对指定的列表进行排序。 12.static void swap(List list, int i, int j)

package com.greedring;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.Collections;
import java.util.Comparator;
import java.util.List;

public class Test {
    public static void main(String[] args) {
        List list = new ArrayList();
        // 1.向集合中添加若干个元素
        Collections.addAll(list, 1, true, "hello", "hello", false, 2, 3);
        System.out.println(list);

        List dest = Arrays.asList(new Object[7]);// java.lang.IndexOutOfBoundsException: Source does not fit in dest
        System.out.println(dest.size());
        // 2.拷贝集合元素
        Collections.copy(dest, list);
        System.out.println("dest=>" + dest);

        // 3.查找指定元素在集合中出现的次数
        int frequency = Collections.frequency(list, "hello");
        System.out.println("hello在list集合中出现的次数:" + frequency);

        // 4.查找集合最大值
        ArrayList l = new ArrayList();
        l.add(1);
        l.add(2);
        l.add(3);
        l.add(4);
        int i = (int) Collections.max(l, new Comparator() {

            @Override
            public int compare(Object o1, Object o2) {
                // TODO Auto-generated method stub
                if (o1 instanceof Integer && o2 instanceof Integer) {
                    Integer i1 = (Integer) o1;
                    Integer i2 = (Integer) o2;
                    if (i1 > i2) {
                        return -1;
                    } else if (i1 < i2) {
                        return 1;
                    } else {
                        return 0;
                    }
                }
                return 0;
            }
        });
        System.out.println("最大值:" + i);
    }
}
