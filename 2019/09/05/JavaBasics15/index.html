线程

多线程概述

进程

正在运行的程序，确切的来说，当一个程序进入内存运行，即是一个或多个进程在运行，具有一定独立功能
计算机分配资源的基本单位

线程

线程是进程中的一个执行单元（执行路径），负责当前进程中程序的执行，一个进程中至少有一个线程。
一个进程中是可以有多个线程的，这个应用程序也可称之为多线程程序。
线程是计算机执行的基本单位。

一个程序运行后至少有一个进程，一个进程中可包含多个线程

单核CPU上多线程就是“假”多线程，在单CPU情况下，某一个时刻，CPU只能执行一个线程
单核CPU：提高资源利用率
多核CPU：提高运行效率

并行：多核CPU同时执行多个任务（多个人同时做不同的事情）
并发：单核CPU（时间片）同时执行多个任务（一个人同时做多件事情）

多线程优点：
  1.提高资源利用率
  2.能够将耗时的操作从主线（main）中移到其他线程中，这样就不会堵塞主线程

什么时候使用多线程：
  1.当同时需要执行多个任务
  2.当主线程遇到耗时操作

一般Java程序运行时会存在三个线程：
main方法运行：系统会默认创建一个主程序(main)
GC：垃圾回收机制线程
异常处理线程

多线程实现

线程创建方式（四种）

第一种 继承Thread类
  • 1.创建普通的类，继承与Thread类
  • 2.重写父类的Thread方法
  • 3.在run方法中白编写线程需要执行的任务代码
  • 4.创建线程对象
  • 5.启动线程 start()
    注意：启动线程调用的是start()方法

Thread相关方法

1.返回当前正在执行的线程对象
static Thread currentThread()

2.返回此线程名称
String getName()

3.改变该线程的名称等于参数(name)
void setName(String name)

4.返回此线程的优先级(CPU调度线程的优先级)
int getPriority()

5.更改此线程的优先级 1(最低)-10(最高)
void setPriority(int newPriority)
线程默认优先级为5：static int 5

6.测试这个线程是否存活
boolean isAlive()

7.在A线程中将B线程加入进来：A线程直接失去CPU执行权，等待B线程执行完成之后，A再执行
void join()

8.当前执行的线程休眠(暂停执行)为指定的毫秒数[1s=1000ms]，根据精度和系统定时器和跳读的准确性
static void sleep(long millis)

9.让线程失去CPU执行权(让步)
static void yield()

    public void run() {
        for (int i = 1; i <= 1000; i++) {
            System.out.println("2..." + Thread.currentThread().getName() + "优先级：" + Thread.currentThread().getPriority() + "，当前输出：" + i);
        }
    }
}

第二种

1.创建一个普通类，实现Runnable接口
2.重写接口中的run方法
3.在run方法中编写任务代码
4.创建实现了Runnable接口的实现类对象
5.将4步中的对象传递给Thread
6.调用start方法启动线程
