线程

多线程概述

进程

正在运行的程序，确切的来说，当一个程序进入内存运行，即是一个或多个进程在运行，具有一定独立功能
计算机分配资源的基本单位

线程

线程是进程中的一个执行单元（执行路径），负责当前进程中程序的执行，一个进程中至少有一个线程。
一个进程中是可以有多个线程的，这个应用程序也可称之为多线程程序。
线程是计算机执行的基本单位。

一个程序运行后至少有一个进程，一个进程中可包含多个线程

单核CPU上多线程就是“假”多线程，在单CPU情况下，某一个时刻，CPU只能执行一个线程
单核CPU：提高资源利用率
多核CPU：提高运行效率

并行：多核CPU同时执行多个任务（多个人同时做不同的事情）
并发：单核CPU（时间片）同时执行多个任务（一个人同时做多件事情）

多线程优点：
  1.提高资源利用率
  2.能够将耗时的操作从主线（main）中移到其他线程中，这样就不会堵塞主线程

什么时候使用多线程：
  1.当同时需要执行多个任务
  2.当主线程遇到耗时操作

一般Java程序运行时会存在三个线程：
main方法运行：系统会默认创建一个主程序(main)
GC：垃圾回收机制线程
异常处理线程

多线程实现

Thread相关方法 （于继承Thread方法创建线程中演示）

1.返回当前正在执行的线程对象
static Thread currentThread()

2.返回此线程名称
String getName()

3.改变该线程的名称等于参数(name)
void setName(String name)

4.返回此线程的优先级(CPU调度线程的优先级)
int getPriority()

5.更改此线程的优先级 1(最低)-10(最高)
void setPriority(int newPriority)
线程默认优先级为5：static int 5

6.测试这个线程是否存活
boolean isAlive()

7.在A线程中将B线程加入进来：A线程直接失去CPU执行权，等待B线程执行完成之后，A再执行
void join()

8.当前执行的线程休眠(暂停执行)为指定的毫秒数[1s=1000ms]，根据精度和系统定时器和跳读的准确性
static void sleep(long millis)

9.让线程失去CPU执行权(让步)
static void yield()

第二种 实现Runnable接口

1.创建一个普通类，实现Runnable接口
2.重写接口中的run方法
3.在run方法中编写任务代码
4.创建实现了Runnable接口的实现类对象
5.将4步中的对象传递给Thread
6.调用start方法启动线程

package com.greedring;

public class Test {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        //通过Runnable接口实现类对象创建Thread对象
        Thread thread = new Thread(myRunnable);//分配一个新的Thread对象  Thread(Runnable target)
        thread.setName("Ring");//改变线程名称
        thread.start();//启动Ring线程

        // 主线程
        for (int i = 1; i <= 1000; i++) {
            System.out.println("1..." + Thread.currentThread().getName() + "..." + i);
        }
    }
}

class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 1; i <= 1000; i++) {
            System.out.println("2..." + Thread.currentThread().getName() + "..." + i);
        }
    }
}

使用匿名内部类创建线程：

 匿名内部类格式：
 new 抽象类/接口(){
     //抽象类、接口中需要重写的方法
 }

解释：创建的是抽象类的子类对象、接口的实现类对象（类是匿名）

package com.greedring;

public class Test {

    public static void main(String[] args) {
        //匿名继承Thread
        new Thread() {
            @Override
            public void run() {
                System.out.println("匿名继承Thread");
            }
        }.start();

        //匿名实现Runnable
        new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println("匿名实现Runnable");
            }
        }).start();

//简写：new Thread(() -> {}).start();
        //单行操作需删除{}
        new Thread(() -> System.out.println("匿名实现Runnable简写")).start();

        //多行操作需要添加{}
        new Thread(() -> {
            for (int i = 1; i <= 1000; i++) {
                System.out.println(i);
            }
        }).start();
    }
}

第三种 通过Callable和FutureTask创建线程

1.创建一个普通类，实现Callable接口
[ V:泛型，线程需要返回什么类型的结果，那么V就指定为什么类型(类名) ]
2.重写接口中的call方法
3.在call方法中编写任务代码
4.创建实现了Callable接口的实现类对象
5.将callable类型对象转换为runnable类型对象
6.将第5步中的runnable对象传递给Thread
7.调用start方法启动线程
8.获取线程返回结果
Callable方式返回线程的返回值需要借助FutureTask（实现了Runnable接口）来接收线程返回值

优点：
  1.能够返回结果
  2.允许方法抛出异常

package com.greedring;

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

public class Test {
    public static void main(String[] args) {
        MyCallable myCallable = new MyCallable();//创建callable实现类对象
        FutureTask<Integer> futureTask = new FutureTask<>(myCallable);//将callable类对象转化为runnable类型对象
        //原因：1.依靠futureTask可以获取线程返回结果 2.将callable类型对象转换为runnable类型对象

        Thread thread = new Thread(futureTask);//启动线程
        thread.start();

        //获取返回值
        try {
            Integer value = futureTask.get();
            System.out.println("1-100的和为：" + value);
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }
}

//实现Callable接口
class MyCallable implements Callable<Integer> {

    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 1; i <= 100; i++) {
            sum += i;
        }
        return sum;
    }
}
线程池创建线程
  • 1.普通类不需要返回值

1.创建普通类实现Runnable接口
2.重写接口中的run方法
3.在run方法中编写任务代码
4.创建连接池对象 (电脑为几核CPU创建几个)
5.创建任务对象
6.提交任务 可以使用submit或者execute
   
   
8.关闭线程(如果有必要)

  • 2.普通类需要含返回值

1.创建普通类实现Callable接口
2.重写接口中的call方法
3.在call方法中编写任务代码
4.创建主函数
5.创建连接池对象 (电脑为几核CPU创建几个)
6.创建任务对象
7.提交任务
8.获取线程返回值
9.关闭线程(如果有必要) 

package com.greedring;

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;

public class Test {
    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService threadPool = Executors.newFixedThreadPool(4);//创建容量为4个线程的线程池

        Task task = new Task();//创建任务

        //提交任务
        Future<String> future1 = threadPool.submit(task);
        Future<String> future2 = threadPool.submit(task);
        Future<String> future3 = threadPool.submit(task);

        //获取线程返回值
        System.out.println(future1.get());
        System.out.println(future2.get());
        System.out.println(future3.get());

        threadPool.shutdown();//关闭线程池

    }
}

class Task implements Callable<String> {

    @Override
    public String call() throws Exception {
        return Thread.currentThread().getName() + "...";
    }
}
