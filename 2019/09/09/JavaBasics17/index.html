集合引入

1.集合：集合（数组）是对多个数据进行存储操作的结构，Java容器

注意：此处的存储是基于内存结构的存储，不同于持久化存储（txt数据库….）

2.数组特点

int[] arr = new int[5];
User[] arr = new User[5];//对象数组
Object[] objects = new Object[5];

  • 数组一旦初始化，长度就固定了（ArrayIndexOutOfBoundsException）
  • 数组一旦定义，那么数组中存储的数据元素的数据类型就确定了

3.数组的缺点：

  • 数组一旦初始化，长度就固定了，无法进行扩容
  • 数组中提供的操作方法过少，删除、插入元素效率过低
  • 数组中没有提供获取数组实际长度的操作
  • 数组中数据默认是有序且可以重复的，如果需要设计无序且不能重复则无法实现

集合概述

1.体系结构：

  • List(子接口) 有序且可重复
     - ArrayList 基于“动态”数组，读取效率高(根据下标、索引)，增删效率慢
     - LinkedList 基于双向链表结构，增删效率高，读取效率慢
     - Vector 基于“动态”数组（线程安全效率低）
  • Set(子接口) 无序不可重复
     - HashSet
     - LinkedHashSet
     - TreeSet


 - HashMap
 - LinkedHashMap
 - TreeMap
 - HashTable
 - Properties

2.Collection 单列集合 常用方法

  • boolean  
    向集合A中添加元素（可选操作）
  • boolean  
    将集合B中的所有元素添加到集合A中（可选操作）
  • void  
    从集合A中移除所有的元素（可选操作）
  • boolean  
    如果集合A包含指定元素，返回true
  • boolean  
    如果集合A包含指定集合B的所有元素，返回true
  • boolean  
    将集合A与集合B进行比较，以进行相等性比较（顺序需要依次对应）
  • int  
    返回集合A的哈希代码值
  • boolean  
    如果集合A不包含任何元素，返回true
  • boolean  
    从集合A中移除指定元素的一个实例，如果它是存在的（可选操作）
  • boolean  
    删除集合A中包含的所有，集合B中的元素（可选操作）
    [当集合A中包含多个相同的元素，该元素在集合B中，全部删除]
  • boolean  
    求两个集合的交集,并将交集返回给集合A
    [如果俩集合相等，返回false]
  • int  
    返回集合A中的元素的数目
  • Object[]  
    返回包含集合A中所有元素的数组

package com.greedring;

import java.util.ArrayList;
import java.util.Collection;
import java.util.Date;
import java.util.Iterator;

public class Test {
    public static void main(String[] args) {
        Collection collection = new ArrayList();
        collection.add(123);
        collection.add("Yae");
        collection.add('a');
        collection.add(123);
        collection.add(new Date());
        collection.add('a');
        System.out.println("创建的集合为："+collection);

        //获取迭代器对象，每次获取默认会存在一个游标，指向集合第一个元素之前的位置
        Iterator iterator = collection.iterator();
        //每次调用next()方法，会实现游标后移，并返回当前游标指向的对象
        Object object = iterator.next();
        System.out.println(object);

        //判断当前游标的下一个是否有元素,存在返回true，不存在返回false
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
        System.out.println("===================================");

        //在遍历时删除元素
        Iterator iterator1 = collection.iterator();
        while (iterator1.hasNext()) {
            Object object1 = iterator1.next();
            if (object1 instanceof Integer && (Integer) object1 == 123) {
                iterator1.remove();//删除当前游标指向的元素
            }
        }
        System.out.println("遍历集合删除123后为："+collection);

        collection.remove('a');//仅删除集合出现的第一个'a'元素
        System.out.println("删除集合中第一次出现的元素'a'后为："+collection);
    }
}

增强for循环

package com.greedring;

import java.util.Arrays;
import java.util.Collection;
import java.util.Date;

public class Test {
    public static void main(String[] args) {
        int[] arr = new int[]{1, 3, 6, 8, 10};
        //普通的for循环，遍历数组的长度0-长度-1，通过下标取值
        for (int i = 0; i < arr.length; i++) {
            System.out.print(arr[i] + "\t");
        }
        System.out.println();

        //增强for循环，可以不需要通过下标取值
        for (int value : arr) {
            System.out.print(value + "\t");
        }
        System.out.println();

        //集合遍历方式，底层也是采用的Iterator的方式
        Collection collection = Arrays.asList(1, "6", true, 'w', new Date());
        for (Object object : collection) {
            System.out.print(object + "\t");
        }
    }
}

可变参数

 1.格式： 数据类型… 变量名  [在形参声明中使用]
 2.可变参数传递实参时，可以传递一个数组的实参（可变参数使用类似数组 length args[i]）
 3.可变参数在调用方法时，可以传递0个、1个、2个…数量的参数
 4.可变参数的方法与同名的方法可以构成重载(但是存在匹配方法时，会优点调用匹配的非可变参数方法)
 5.可变参数的方法与同名的数组作为参数的方法不构成重载
 6.可变参数必须声明在形参列表的最后一个
 7.在一个方法的参数列表中，只能声明一个可变参数

package com.greedring;

public class Test {
    public static void main(String[] args) {
        test();
        test(1);
        test(1, 2);
        test(1, 2, 3);
    }

    private static void test() {
        System.out.println("无参函数被调用");
    }

    private static void test(int a) {
        System.out.println(a + "...test");
    }

    private static void test(int a, int b) {
        System.out.println(a + "...test..." + b);
    }

    // 用可变参数（结构类似数组）代替数组
    private static void test(int... args) {
        System.out.println("可变参数函数被调用了");
        for (int arg : args) {
            System.out.print(arg + "\t");
        }
    }
}

以HashSet为例
无序性：数据在存储时并非按照数组索引顺序进行存储的，是由存储数据的哈希值决定的
不可重复性：在添加数据时调用hashCode、equals方法，返回值不能为true。

HashSet添加元素底层实现原理

在HashSet中添加元素a，首先调用元素a对象的hashCode方法获取七哈希值
根据哈希值通过某种算法，找到需要在数组中存储的位置（引索），如果该位置还没有存储元素，那么元素a添加成功（情况1）
如果该位置存储了元素b，那么就比较两者的哈希值是否相等，如果两种哈希值一致，那么元素a添加成功

Hash底层实现原理
HashMap map = new HashMap();
当创建HashMap时，底层默认创建一个Entry数组，并进行默认初始化，初始化长度为16

//之前已经存储了部分key…value
map.put(k1,v1)
先调用k1对象的hashCode方式，，根据k1的哈希值通过某种算法，得到元素需要再底层数组存储的位置（索引0-）
判断当前索引位置上是否有元素，如果当前索引位置上没有元素，k1，v1直接添加成功[情况1]
判断当前索引位置上存在一个或多个元素，将k1的哈希值依次与这一个或多个已经存在的key的哈希值进行比较，如果k1的哈希值与已存在元素的哈希值都不同，k1、v1添加成功[情况2]
如果k1的哈希值与已经存在的元素中的任意一个元素的key的哈希值相同(k2,v2)
调用equals方法比较k1、k2，如果返回false，k1、v1添加成功[情况3]
如果equals方法返回true，那么v1值将覆盖v2[情况4]

1.对于情况23多个元素再同一个数组中的位置使用链表进行存储
2.对于put过程（添加元素），可能会涉及到扩容操作，当数组使用长度超过临界值（而并非是数组长度），并且当前数组位置不为空
