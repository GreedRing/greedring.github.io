集合引入

1.集合：集合（数组）是对多个数据进行存储操作的结构，Java容器

注意：此处的存储是基于内存结构的存储，不同于持久化存储（txt数据库….）

2.数组特点

int[] arr = new int[5];
User[] arr = new User[5];//对象数组
Object[] objects = new Object[5];
(1).数组一旦初始化，长度就固定了（ArrayIndexOutOfBoundsException）
(2).数组一旦定义，那么数组中存储的数据元素的数据类型就确定了

3.数组的缺点：

(1).数组一旦初始化，长度就固定了，无法进行扩容
(2).数组中提供的操作方法过少，删除、插入元素效率过低
(3).数组中没有提供获取数组实际长度的操作
(4).数组中数据默认是有序且可以重复的，如果需要设计无序且不能重复则无法实现

集合概述

1.体系结构：

Collection：单列集合（一个个数据组成）

  • List(子接口) 有序且可重复
     - ArrayList 基于“动态”数组，读取效率高(根据下标、索引)，增删效率慢
     - LinkedList 基于双向链表结构，增删效率高，读取效率慢
     - Vector 基于“动态”数组（线程安全效率低）

  • Set(子接口) 无序不可重复
     - HashSet
     - LinkedHashSet
     - TreeSet

Map：双列集合（一对对数据组成）
 - HashMap
 - LinkedHashMap
 - TreeMap
 - HashTable
 - Properties

2.Collection 单列集合 常用方法

  • 向集合A中添加元素（可选操作）
    boolean 集合A.add(E e);

  • 将集合B中的所有元素添加到集合A中（可选操作）
    boolean 集合A.addAll(集合B);

  • 从集合A中移除所有的元素（可选操作）
    void 集合A.clear();

  • 如果集合A包含指定元素，返回true
    boolean 集合A.contains(Object o);

  • 如果集合A包含指定集合B的所有元素，返回true
    boolean 集合A.containsAll(集合B);

  • 将指定的对象与集合A进行比较，以进行相等性比较

??????????????
boolean 集合A.equals(Object o);

  • 返回集合A的哈希代码值
    int 集合A.hashCode();

  • 如果集合A不包含任何元素，返回true
    boolean 集合A.isEmpty();

  • 从集合A中移除指定元素的一个实例，如果它是存在的（可选操作）
    boolean remove(Object o)

  • 删除集合A中包含的所有，集合B中的元素（可选操作）
    boolean 集合A.removeAll(集合B);
    [当集合A中包含多个相同的元素，该元素在集合B中，全部删除]

================================

  • 求两个集合的交集,并将交集返回给当前集合
    boolean retainAll(Collection<?> c)

  • 返回此集合中的元素的数目。
    int size()

  • 返回包含此集合中所有元素的数组。
    Object[] toArray()

  • 返回包含此集合中所有元素的数组；返回数组的运行时类型是指定的数组的运行时类型
    T[] toArray(T[] a)

  • 返回一个可能并行 Stream与集合的来源。
    Iterator iterator()
