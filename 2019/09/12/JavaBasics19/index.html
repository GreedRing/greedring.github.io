day19

Map

接口，双列集合（存储一对一对的数据）类似函数y=f(x)

HashMap

Map的主要实现类，线程不安全，效率高，可存储为null的key和value
LinkedMapHashMap的子类，能够安装元素的添加顺序进行遍历（因为每个元素包含了两个引用）
如果涉及到频繁遍历操作时，LinkedHashMap性能高于HashMap

TreeMap

可以对添加到集合中的元素进行排序（按照key的自然排序或者如果key是对象的话，

Hashtable

古老的双列集合，线程安全，效率低
Properties操作属性配置文件的key和value都必须是String型

Map、key、value的特点
以HashSet为例
Key是无序且不可重复，如果需要获取一个Map的所有的Key，可以使用的是Set结构
Value是无序可重复的，如果需要获取一个Map的所有Value，可以使用Collection结构存储
一个Map中的Entry对象无序且不可重复的，如果需要获取一个Map的所有键值对(Entry)，可以使用Set结构

Hash底层实现原理
HashMap map = new HashMap();
当创建HashMap时，底层默认创建一个Entry数组，并进行默认初始化，初始化长度为16

//之前已经存储了部分key…value
map.put(k1,v1)
先调用k1对象的hashCode方式，，根据k1的哈希值通过某种算法，得到元素需要再底层数组存储的位置（索引0-）
判断当前索引位置上是否有元素，如果当前索引位置上没有元素，k1，v1直接添加成功[情况1]
判断当前索引位置上存在一个或多个元素，将k1的哈希值依次与这一个或多个已经存在的key的哈希值进行比较，如果k1的哈希值与已存在元素的哈希值都不同，k1、v1添加成功[情况2]
如果k1的哈希值与已经存在的元素中的任意一个元素的key的哈希值相同(k2,v2)
调用equals方法比较k1、k2，如果返回false，k1、v1添加成功[情况3]
如果equals方法返回true，那么v1值将覆盖v2[情况4]

1.对于情况23多个元素再同一个数组中的位置使用链表进行存储
2.对于put过程（添加元素），可能会涉及到扩容操作，当数组使用长度超过临界值（而并非是数组长度），并且当前数组位置不为空
